# PGA Fantasy Model — Claude Context

## Project Overview
A self-contained PGA Tour fantasy golf projection model and interactive HTML dashboard. Hosted via GitHub Pages at `https://botsrunmylife.github.io/PGA/`. Repo: `Botsrunmylife/PGA`.

## File Structure
```
index.html       — Dashboard UI + model engine (CSS, JS, HTML all-in-one)
data.js          — Player & event data (auto-generated by fetch_data.py)
fetch_data.py    — Python script that pulls real stats from ESPN public API
CLAUDE.md        — This file
```

## How the Model Works
7 weighted scoring factors, all adjustable via sliders in the sidebar:
1. **SG: Total** — raw strokes gained baseline
2. **SG: Splits (event-weighted)** — OTT/APP/ATG/Putt weighted by course emphasis
3. **Recent Form (10 events)** — recency-weighted inverse scoring with trend detection
4. **Course History** — actual past finishes at each venue, best-finish-weighted
5. **Course Fit Profile** — dot-product match of player SG profile to course demands
6. **Ballstriking Stats** — GIR%, driving distance, accuracy composite
7. **Short Game & Scrambling** — scramble%, SG:ATG, SG:Putting composite

Field strength adjustment applied for smaller fields (Masters 87, Signature 120).

## Data Flow
- `fetch_data.py` hits ESPN's free public API (no key needed):
  - `sports.core.api.espn.com/v2/sports/golf/...` endpoints
  - Season stats (driving, GIR, scrambling, scoring avg, birdies)
  - Event-by-event results across 2024-2026 seasons
  - Course history at 8 tracked events
- SG splits are **estimated** from available ESPN stats (ESPN doesn't provide true SG)
- DK salaries are **estimated** from performance — user should paste real DK CSV values
- `data.js` uses full key names (`name`, `course`, `type`) — `index.html` has a data adapter that maps to short keys (`nm`, `cr`, `tp`) used internally

## Key ESPN Player IDs
Correct IDs are in `fetch_data.py` TOP_PLAYER_IDS list. They were verified via ESPN search API. Jon Rahm (9780) skips because he's on LIV Golf with no PGA Tour stats.

## Known Issues / Gotchas
- ESPN sometimes returns zero-value stats for players with few 2026 events (e.g., Sungjae Im, Justin Thomas, Will Zalatoris). The fetcher has zero-guards but SG estimates may be off for them.
- The fetcher takes ~15 minutes due to rate-limited API calls (~0.3s delay between requests, many requests per player for course history).
- Tooltips use `data-tip` attribute with CSS `::after` pseudo-element, NOT the `title` attribute (which shows the ugly browser default).
- The EVENTS object key format differs between hand-written data.js (short: `nm`, `cr`, `tp`) and auto-generated data.js (full: `name`, `course`, `type`). The adapter in index.html handles both.
